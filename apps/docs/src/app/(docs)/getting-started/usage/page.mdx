    # Basic Usage

    This section provides a basic overview of how to use the XStack Library components.

    ## `xstack-table`

    The `xstack-table` component is designed to display tabular data with features like searching and pagination.

    ### Example

    ```html
    <xstack-table
      .data="${JSON.stringify([{ id: 1, name: 'Item 1' }, { id: 2, name: 'Item 2' }])}"
      .columns="${JSON.stringify([{ key: 'id', header: 'ID' }, { key: 'name', header: 'Name' }])}"
    ></xstack-table>
    ```

    ### Properties

    -   `data`: (Array) The array of data objects to display in the table. (e.g., `[{ id: 1, name: 'Test' }]`)
    -   `columns`: (Array) An array of column definitions. Each object should have a `key` and `header`. (e.g., `[{ key: 'id', header: 'ID' }]`)
    -   `pageSize`: (Number) The number of items to display per page. Default is 10.

    ### Events

    -   `search`: Fired when the search input value changes. The event detail contains the search query.
    -   `pageChange`: Fired when the current page changes. The event detail contains the new page number.

    ## `xstack-modal`

    The `xstack-modal` component provides a flexible modal dialog.

    ### Example

    ```html
    <xstack-modal .isOpen="${true}" @close="${() => console.log('Modal closed')}">
      <div slot="header">Modal Title</div>
      <div slot="body">
        <p>This is the content of the modal body.</p>
      </div>
      <div slot="footer">
        <button>Close</button>
      </div>
    </xstack-modal>
    ```

    ### Properties

    -   `isOpen`: (Boolean) Controls the visibility of the modal. Set to `true` to show, `false` to hide.

    ### Events

    -   `close`: Fired when the modal is requested to be closed (e.g., by clicking the close button or overlay).

    ### Slots

    -   `header`: Content for the modal header.
    -   `body`: Content for the modal body.
    -   `footer`: Content for the modal footer.
